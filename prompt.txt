GitHub Issue #181: feat: double-click to edit, single-click to select, D to delete

Description:
## What
Change calendar event interaction: single-click selects an event (visual highlight), double-click opens the edit popover. Pressing `D` while an event is selected deletes it immediately.

## Why
Currently single-click opens the edit menu immediately, making it too easy to accidentally trigger. Deleting requires two steps (edit menu → delete button → confirm popup). This adds unnecessary friction to common operations.

## Files to touch
- `src/app/(dashboard)/calendar/page.tsx` — change `onEventClick` handler to set a `selectedEventId` state instead of dispatching `OPEN_POPOVER`; add `onEventDoubleClick` handler that dispatches `OPEN_POPOVER`; add a `useEffect` with a `keydown` listener for `D` that calls `store.deleteEvent(selectedEventId)` when an event is selected
